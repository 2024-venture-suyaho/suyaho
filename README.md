## 💡 프로젝트 소개
수야호는 '지역사회 문제 해결: 환경문제 해결'을 주제로 하여 물건의 재사용과 잃어버린 물건의 주인 찾기를 통해 새로운 제품 구매 시 발생하는 불필요한 일회용품과 포장재 사용을 줄이기 위해 제작된 프로젝트입니다.

#### * 주제 선정 이유 *
중고거래 시장이 활성화됨에 따라 삼육대학교 학생들도 중고 물품 거래를 통해 원하는 물품을 효율적으로 얻거나 처분(판매 혹은 나눔)하려고 노력하고 있습니다. 그러나 기존의 물품 거래 과정에는 여러 불편함이 존재합니다.

1. 특화된 플랫폼의 부재: 소규모 학교 커뮤니티를 대상으로 한 특화된 플랫폼이 없어 거래 과정이 불편하고 안정성에 대한 우려가 있습니다.
2. 검색의 어려움: 대학생 커뮤니티를 이용하다 보면 수업 내용 질문과 겹쳐 원하는 물품을 검색하는 데 어려움을 겪습니다.
3. 게시글의 혼잡: 학교 커뮤니티에서는 분실물을 찾거나 찾아가라는 게시글이 많이 올라오지만, 이러한 게시글이 분류되지 않고 무분별하게 올라와 원하는 물품을 찾기가 어렵습니다.
4. 게시글의 분류 부족: 커뮤니티에서는 가끔 기부(무료나눔, 나눔, 필요하신 분) 게시물이 올라오지만, 일반 게시글과 함께 작성되어 필요한 사람들이 해당 물품의 나눔을 받지 못하는 경우도 종종 있습니다.<br>
- 따라서 수야호는 이러한 문제를 해결하고자 원하는 품목을 편하게 찾아보기 위한 플랫폼을 서비스하기위해 기획되었습니다.

#### * 프로그램의 영향 *
수야호는 과거 90년대 유행했던 환경운동 *아나바다(아껴 쓰고, 나눠 쓰고, 바꿔 쓰고, 다시 쓰기)*를 모티브로 하여 만든 게시판입니다. 수야호는 다음과 같은 방식으로 환경 문제 해결에 기여하고자 합니다

- 아껴 쓰기: 물건을 오래 사용하여 새로운 물건을 구매하는 빈도를 줄입니다.<br>
- 나눠 쓰기: 필요한 물건을 무료로 나누어주거나 교환합니다.<br>
- 바꿔 쓰기: 사용하지 않는 물건을 다른 사람과 교환하여 자원의 낭비를 줄입니다.<br>
- 다시 쓰기: 중고거래를 통해 물건을 재사용하고 자원의 순환을 촉진합니다.<br>

**기대 효과**
- 환경 보호: 자원 절약과 폐기물 줄이기에 기여합니다. 중고거래를 통해 물건을 재사용하면 원자재와 에너지를 절약하고, 폐기물을 줄일 수 있습니다.
- 경제적 효율성: 학생들이 저렴한 가격에 필요한 물건을 구입할 수 있어 경제적 부담을 줄일 수 있습니다.
- 커뮤니티 강화: 분실물 찾기, 나눔, 교환을 통해 학생들 간의 협력하는 문화를 조성합니다.
- 편리한 거래: 특화된 플랫폼을 통해 원하는 물품을 쉽게 검색하고 거래할 수 있어 편리한 거래 환경을 제공합니다.
- 수야호는 이러한 목표를 통해 삼육대학교 커뮤니티 내에서 환경문제 해결과 더불어 학생들의 삶의 질을 향상시키는 데 기여하고자 합니다.

## 📌 브랜치 활용

- **main 브랜치:** 최종 코드 관리 브랜치로, prototype 브랜치에서 문제가 없으면,merge 해서 사용
- **feature 브랜치:** 각자 담당한 기능 단위로 브랜치를 나누어서 사용
- **prototype 브랜치:** merge한브렌치

## ⚙ 프로그램 주요 기능
- 로그인/회원가입 : 이름, 전화번호, 학과, 이메일, 비밀번호, 비밀번호 확인 후에 회원가입을 진행합니다. 회원가입을 한 정보는 데이터베이스에 저장되고, 그 정보를 통해 로그인 할 수 있습니다.  
- 거래게시판 : 거래게시판이 보이며 거래게시판에 들어가면 현재 거래하고자하는 목록을 볼수있습니다. 목록에는 번호, 제목, 카테고리, 등록일시, 아이디, 거래여부 를 보여주고, 위에있는 검색창을 이용하여 원하는 항목을 검색할수도있고 거래게시판 검색버튼 오른쪽에있는 쓰기를 들어가면 게시물을 쓸수있습니다.
- 마이페이지 : 회원가입할떄 입력한 전화번호, 비밀번호, 전과시 학과를 변경할수있습니다. 그리고 프로필페이지를 변경할수있습니다.<br> 그리고 본인이 작성한 게시글 목록을 볼수있고, 거래상태를 수정할수있습니다 (거래전, 거래중, 거래완료) 그리고 거래게시판에서 글을 삭제할수있습니다.
- 관리자페이지 : 관리자페이지는 관리자-유저보기, 관리자-게시글보기 두가지 탭으로 나뉩니다. 관리자-유저보기는 유저의 상세정보를 열람할수있고, 이름, 학과, 학번으로 유저를 검색할 수 있으며 유저를 삭제하여 회원탈퇴 처리를 할 수 있습니다. 관리자-게시글 보기에서는 게시글 목록을 열람할수있으며, 학번, 제목으로 게시글을 검색할 수 있으며 게시글을 삭제시킬 수있습니다.

## 🙏 개발 후기
#### [ 팀장_심선우 ]
Spring으로 하는 첫 프로젝트였는데, 지금까지 벤처에서 했던 프로젝트 중 가장 흥미롭고 재미있게 참여했던 프로젝트였습니다.
특히 Spring의 구조를 많이 생각하며 고민하고 행해서 그런지, 이제는 나만의 코드 작성 구조와 순서가 어느정도 구체화되었다는 면에서 좋았습니다.
프로젝트가 생각보다 단기라서 시간에 쫓기듯 하고, 그렇기에 팀원 모두가 조금은 예민한 상태였지만 한 번씩 부딪히면서 협업의 중요성을 더 깨닫게 되었습니다.

#### [ 팀원_김현수 ]
백엔드 개발에서는 Spring Boot와 JPA를 사용하여 서버 측 로직을 구현하고, 데이터베이스와의 연동을 경험하였습니다.너무 어렵고 잘 되지 않아 힘들었지만프로젝트를 진행하면서 많은 것을 배우고 성장할 수 있었습니다.

프로젝트를 성공적으로 마무리하면서 자신감도 얻을 수 있었습니다. 처음에는 막막하고 어려웠던 부분들도 끝까지 포기하지 않고 해결해나가는 과정에서 성취감을 느꼈습니다. 앞으로도 계속 문제를 해결해나가며 계속 성장하는 개발자가 되고싶다고 생각하였습니다

#### [ 팀원_오송은 ]
프론트와 백엔드 영역을 연동하는 프로젝트를 처음 진행하며 일주일 안에 끝내려고 하니 힘든 부분도 많았지만 직접 구현해보며
그만큼 관련 지식에 대해 더 많은 것을 쌓게되는 프로젝트가 된 것 같아 뿌듯했습니다. 


#### [ 팀원_이은서 ]
전부터 배우고싶던 스프링을 사용해서 게시판을 만들어보았습니다. 처음 사용해보는 언어이기도 하고 데이터베이스
연동하여 사용하여 개발을 해서 구현이 어려웠지만 그래도 제가 만든 파트는 정상적으로 작동하는것을 보고 뿌듯하였습니다.


#### [ 팀원_이재혁 ]
이번엔 하고 싶었던 채팅기능을 구현해볼려고 주말부터 자료찾으면서 진행을 했습니다. 생각보다 레퍼런스가 다양해 어떤 방식으로 해야 내가 완성할수 있고 지금 플젝에 적용할수 있을지 고민하다
텍스트메시지웹소켓이라는 인터페이스 클래스를 이용해 실시간 메세지 소통을 해볼려고 했습니다..... 하지만 생각보다 마음대로 잘 안돼서.... 소켓까지 웹사이트에 연동시키고 서로 대화가 가능하게 만들었지만
DB에서 아이디를 받아서 1:1 채팅을 구현하는데 막혀 결국 완성은 하지 못하고 시도로 끝났습니다 그래도 여태까지 뭔가 해보고 싶다는 기능보다 해야한다 기능을 많이 했었는데 처음으로 해보고 싶었던 기능을 해서
충분히 공부도 된것 같아 나름의 성장할 수 있는 계기가 된것 같습니다. 담에 비슷한 기능을 구현한다면 연동에 시간을 소비하는게 아니라 어떻게 채팅을 플젝에 맞춰 잘 기능하게 할건지 더 시간을 쏟을수 있을것 같습니다.
